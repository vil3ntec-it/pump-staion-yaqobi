# دستورهای دائمی این ریپو (برای Claude)

این دستورها یک‌بار توسط صاحب ریپو داده شده و برای همهٔ سیزن‌های بعدی معتبر است:

## جریان کار اتومات — بدون سوال از کاربر

بعد از تمام شدن هر تغییر (تاییدشده در تاریخ 2026-07-13 توسط صاحب ریپو):

1. تغییرات را روی برنچ کاری کامیت و پوش کن.
2. **اتومات Pull Request به `main` بساز** — بدون این که کاربر بگوید.
3. **اتومات PR را مرج کن (squash)** — بدون این که کاربر بگوید.
4. دیپلوی خودکار است: با مرج شدن به `main`، ورک‌فلو `deploy-pages.yml` سایت را روی GitHub Pages می‌برد.

فقط وقتی مرج نکن که تغییر ناقص/خراب باشد یا خود کاربر در همان سیزن چیز دیگری گفته باشد.

## نسخه‌گذاری

در هر تغییر، نسخه را یک واحد بالا ببر — هر دو جا با هم:

- `index.html` → `const APP_VERSION = '2.9.x';`
- `sw.js` → `const CACHE_NAME = 'pump-yaqobi-shell-v2.9.x';`

اگر این دو هم‌نسخه نباشند، Service Worker نسخهٔ کهنه را از کش سرو می‌کند و کاربر تغییرات را نمی‌بیند.

## سبک کامیت و PR

- پیام کامیت و عنوان PR به فارسی، مثل کامیت‌های قبلی، با ذکر شمارهٔ نسخه در پرانتز:
  «... (نسخه 2.9.x)»

## نکته‌های فنی پروژه

- کل برنامه یک فایل است: `index.html` (HTML + CSS + JS داخلی) — PWA با `sw.js` و `manifest.json`.
- منطق حساب قرض‌داران (مودال شخص) حساس است: «واحد پول» و «واحد تیل» دو دفتر کاملاً جدا هستند (`rows` و `moneyRows`). جمع‌های نمایشی نباید روی محاسبات الباقی اثر بگذارند.
- برای تست دستی: صفحه با قفل رمز باز می‌شود؛ در تست headless می‌توان `lockScreen` را مخفی و `app` را نمایان کرد.
