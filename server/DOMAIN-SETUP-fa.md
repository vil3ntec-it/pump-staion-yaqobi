# دامنهٔ ثابت + تونل نام‌دار کلادفلر (حلِ همیشگیِ «سرور هر بار عوض می‌شود»)

این راهنما مشکلِ «اطلاعات برای همهٔ دیوایس‌ها نمی‌آید» را **برای همیشه** حل می‌کند.

## چرا الان قطع می‌شود؟
آدرس فعلی سرور از نوع **Quick Tunnel** رایگان است (`xxxx.trycloudflare.com`).
این آدرس **هر بار که سرور یا cloudflared را ری‌استارت کنی عوض می‌شود** — برای همین
سایت به آدرس مُردهٔ قدیمی وصل می‌ماند و هیچ دیوایسی سینک نمی‌شود.

راه‌حل: یک **دامنهٔ ثابت** + **تونل نام‌دار (Named Tunnel)**. آدرس مثل
`wss://sync.yourdomain.xyz` می‌شود و **هیچ‌وقت عوض نمی‌شود** — سرور را هزار بار هم
خاموش‌روشن کنی، همهٔ دیوایس‌ها وصل می‌مانند و IP خانه‌ات هم کاملاً مخفی است.

---

## گام ۱ — خرید دامنه
یک دامنهٔ ارزان بخر (سالی حدود ۱ تا ۳ دلار):
- **بهترین جا: خودِ Cloudflare** → منوی **Domain Registration › Register Domains**.
  مزیت: دامنه مستقیم داخل کلادفلر می‌آید و لازم نیست nameserver عوض کنی.
- جایگزین: **Namecheap** یا **Porkbun** (بعداً باید دامنه را به کلادفلر Add کنی).

نکته‌ها:
- پسوند ارزان: `.xyz` یا `.online` یا `.site`.
- چون قرار است «کسی نبیند»، یک اسم بی‌ربط بگیر (مثلاً `pmp-sync-9x.xyz`). قشنگ بودنش مهم نیست.
- **کارت بانکی بین‌المللی (Visa/Mastercard) لازم است.** اگر نداری، Porkbun گاهی
  گزینه‌های پرداخت بیشتری دارد.

## گام ۲ — دامنه را داخل Cloudflare بیاور
- اگر از **Cloudflare** خریدی: خودکار آنجاست، این گام تمام است.
- اگر جای دیگر خریدی: در داشبورد Cloudflare روی **Add a site** بزن، دامنه را وارد کن،
  پلن **Free** را انتخاب کن، و **دو تا nameserver** که کلادفلر می‌دهد را در پنل
  فروشندهٔ دامنه (Namecheap/Porkbun) جای‌گزین کن. (چند دقیقه تا چند ساعت طول می‌کشد.)

## گام ۳ — نصب cloudflared و لاگین (روی همان کامپیوتر سرور)
1. فایل `cloudflared-windows-amd64.exe` را از
   <https://github.com/cloudflare/cloudflared/releases> بگیر و اسمش را بگذار `cloudflared.exe`.
2. در Command Prompt بزن:
   ```
   cloudflared tunnel login
   ```
   مرورگر باز می‌شود؛ دامنه‌ات را انتخاب و **Authorize** کن.

## گام ۴ — ساخت تونل نام‌دار و وصل به دامنه
```
cloudflared tunnel create pump
cloudflared tunnel route dns pump sync.yourdomain.xyz
```
- به‌جای `yourdomain.xyz` دامنهٔ واقعی‌ات را بگذار.
- `sync` هر زیردامنه‌ای می‌تواند باشد (مثلاً `s`, `api`, `db`). همان که انتخاب کردی را
  یادت باشد — می‌شود بخشی از آدرس نهایی.
- دستور اول یک **Tunnel ID** و یک فایل `.json` می‌سازد (مسیرش را چاپ می‌کند).

## گام ۵ — فایل تنظیمات تونل
یک فایل به اسم `config.yml` در مسیر `C:\Users\<نام‌کاربری>\.cloudflared\` بساز با این محتوا:
```yaml
tunnel: pump
credentials-file: C:\Users\<نام‌کاربری>\.cloudflared\<TUNNEL-ID>.json
ingress:
  - hostname: sync.yourdomain.xyz
    service: http://localhost:8787
  - service: http_status:404
```
- `<TUNNEL-ID>` همان چیزی است که در گام ۴ چاپ شد (اسم فایل `.json`).
- پورت `8787` همان پورت سرور شخصی است (اگر عوضش نکرده‌ای، دست نزن).

## گام ۶ — اجرا
اول سرور را بالا بیاور (`start-windows.bat` یا `node server.js`)، بعد تونل:
```
cloudflared tunnel run pump
```
تست: در مرورگر برو به `https://sync.yourdomain.xyz/health` — باید `{"ok":true,...}` ببینی.

### (اختیاری ولی توصیه‌شده) اجرای خودکار با روشن‌شدن ویندوز
تا لازم نباشد هر بار دستی اجرا کنی:
```
cloudflared service install
```
این تونل را به‌صورت سرویس ویندوز نصب می‌کند و با هر روشن‌شدن خودکار بالا می‌آید.
(سرورِ `node` را هم می‌توانی در Startup ویندوز بگذاری تا خودکار اجرا شود.)

## گام ۷ — آدرس نهایی را به Claude بده
آدرس ثابتت این است (فقط `https` را به `wss` تبدیل کن):
```
wss://sync.yourdomain.xyz
```
این آدرس + **رمز سرور** (همان که سرور موقع اجرا چاپ می‌کند) را به Claude بده تا:
- در `index.html` مقدار `SELF_HOST_URL` را با همین آدرس ثابت جای‌گزین کند،
- نسخه را بالا ببرد و سایت را دیپلوی کند.

از آن به بعد، همهٔ دیوایس‌ها **همیشه** به سرور تو وصل‌اند و اطلاعات برای همه می‌آید —
دیگر هیچ‌وقت لازم نیست دستی آدرس را عوض کنی. ✅
