# سرور شخصی پمپ یعقوبی (جایگزین فایربیس)

این سرور کوچک، **جای فایربیس** را می‌گیرد: همهٔ داده‌ها، بکاپ‌ها، چت و «حضور آنلاین»
به‌جای سرور گوگل، روی **کامپیوتر خودت** ذخیره می‌شوند و محدودیت فضای فایربیس برداشته می‌شود.

## مهم‌ترین ویژگی: خیلی ساده است
- ❌ نیازی به نصب PostgreSQL **نیست**.
- ❌ نیازی به `npm install` **نیست** (کتابخانهٔ لازم همراه همین پوشه است).
- ✅ فقط **Node.js** لازم است. داده در پوشهٔ `data` (فایل‌های ساده) ذخیره می‌شود.

---

## گام ۱ — دیگر لازم نیست Node.js را دستی نصب کنی! 🎉
کافی است روی **`start-windows.bat`** دوبار کلیک کنی. اگر Node.js روی کامپیوترت نبود،
**خود اسکریپت نسخهٔ «قابل‌حمل» Node را خودکار دانلود می‌کند** (بدون نصب، بدون دسترسی مدیر،
فقط یک‌بار، داخل پوشهٔ `node-portable`). بعد از دانلود، سرور خودش بالا می‌آید.

> فقط **اینترنت** لازم است تا این دانلودِ یک‌بارهٔ حدود ۳۰ مگابایتی انجام شود. دفعات بعد
> دیگر دانلود نمی‌کند و مستقیم اجرا می‌شود.

اگر خودت از قبل Node.js نصب داری، همان استفاده می‌شود. اسکریپت Node را در همهٔ این حالت‌ها
هم پیدا می‌کند: نصب رسمی، winget، nvm، scoop، chocolatey، fnm و نصب «فقط برای این کاربر».

## گام ۲ — اجرای سرور
پوشهٔ `server` را جایی مثل Desktop بگذار، بعد یکی از این دو کار:

**راه ساده:** روی فایل **`start-windows.bat`** دوبار کلیک کن (اگر پیام امنیتی آمد، Run).

**یا در Command Prompt** (داخل پوشهٔ server):
```
node server.js
```

سرور بالا می‌آید و یک **«رمز سرور»** چاپ می‌کند (چیزی مثل `35ca3d...`).
این رمز را نگه دار — در برنامه لازمش داری.

تست سلامت: در مرورگر برو به <http://localhost:8787/health> — باید `{"ok":true,...}` ببینی.

> اگر `node.exe` را در مسیر غیرمعمولی نصب کرده‌ای و `start-windows.bat` پیدایش نکرد،
> در cmd داخل پوشهٔ server این را بزن (مسیر خودت را بگذار):
> ```
> "F:\مسیر\node.exe" server.js
> ```

## گام ۳ — وصل کردن برنامه به سرور
در برنامه برو به: **تنظیمات › هم‌زمان‌سازی › سرور شخصی** و پر کن:
- **آدرس سرور:** فعلاً برای تست روی همین کامپیوتر: `ws://localhost:8787`
- **رمز سرور:** همان رمزی که سرور چاپ کرد.

دکمهٔ «ذخیره و اتصال» را بزن. تمام — برنامه روی سرور خودت کار می‌کند.

## در دسترس کردن از اینترنت (اختیاری — Cloudflare Tunnel رایگان)
اگر می‌خواهی از بیرونِ خانه هم وصل شوی:
1. از <https://github.com/cloudflare/cloudflared/releases> فایل `cloudflared-windows-amd64.exe` را بگیر و اسمش را بگذار `cloudflared.exe`.
2. در Command Prompt بزن:
   ```
   cloudflared tunnel --url http://localhost:8787
   ```
3. یک آدرس مثل `https://xxxx.trycloudflare.com` می‌دهد. در برنامه به‌جای `https` بنویس **`wss`**:
   `wss://xxxx.trycloudflare.com`
   و همان را در فیلد «آدرس سرور» بگذار.

> ⚠️ **هشدار مهم:** این آدرس رایگان (`trycloudflare.com`) **هر بار که سرور یا cloudflared
> را ری‌استارت کنی عوض می‌شود** — برای همین بعد از مدتی سینک برای همهٔ دیوایس‌ها قطع می‌شود.
> برای حلِ **همیشگیِ** این مشکل (آدرس ثابت که هیچ‌وقت عوض نمی‌شود) راهنمای
> **[`DOMAIN-SETUP-fa.md`](./DOMAIN-SETUP-fa.md)** را ببین: یک دامنهٔ ارزان + تونل نام‌دار.

## نکته‌ها
- **کامپیوتر باید روشن باشد** تا سرور در دسترس باشد. تغییرات آفلاینِ کاربران گم نمی‌شود؛ به‌محض روشن‌شدن دوباره و وصلِ نت، خودکار ذخیره می‌شود.
- **بکاپ:** کل داده در پوشهٔ `data` است. برای بکاپ، فقط از پوشهٔ `data` یک کپی بگیر.
- **رمز سرور** در `data/token.txt` است؛ آن را به کسی نده.

## تست سریع (بدون مرورگر)
با سرورِ روشن، در پوشهٔ server:
```
node test-client.mjs
```
باید ۱۲ تستِ سبز ببینی.
