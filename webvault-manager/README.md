# 🔐 WebVault Manager

اپلیکیشن **مدیریت متمرکز سایت‌ها، دامنه‌ها، سرورها، رمزها، بکاپ‌ها و مشتری‌ها** برای کسی که چندین سایت طراحی و فروخته و همه‌چیز را روی یک **سرور خانگی** نگه می‌دارد.

اطلاعات روی سرور خودتان ذخیره می‌شود (نه هیچ ابری)، همهٔ مقادیر حساس با **AES-256-GCM** رمزنگاری می‌شوند، و برنامه با یک **رمز اصلی (Master Password)** باز می‌شود.

---

## چرا این معماری؟ (مهم)

درخواست اولیه Electron + React + TypeScript + PostgreSQL بود. اما این ریپو یک واقعیت مهم دارد: **سرور خانگی شما فقط Node.js دارد — بدون ابزار build، بدون `npm install`، بدون دیتابیس جداگانه** (دقیقاً مثل سرور فعلی برنامهٔ پمپ). یک استک React/TS/Vite/native-SQLite در آن محیط اصلاً اجرا نمی‌شود.

به‌عنوان یک تصمیم مهندسی، چیزی ساخته شده که **همین حالا و بدون نصب هیچ‌چیز اجرا می‌شود**:

| بخش | فناوری | چرا |
|-----|--------|-----|
| **Backend** | فقط Node.js ≥ ۲۲ (بدون وابستگی) | روی سرور خانگی بدون `npm install` اجرا می‌شود |
| **Database** | **SQLite داخلی Node** (`node:sqlite`) | دیتابیس واقعی SQL، بدون کامپایل و بدون نصب |
| **رمزنگاری** | `node:crypto` — AES-256-GCM + scrypt | استاندارد صنعتی، بدون کتابخانهٔ بیرونی |
| **Frontend** | SPA حرفه‌ای Dark-Mode (بدون build) | سرور همان را سرو می‌کند؛ در مرورگر یا داخل Electron باز می‌شود |
| **Desktop App** | پوستهٔ **Electron** (اختیاری) | «اپ دسکتاپ» واقعی که به سرور خانگی وصل می‌شود؛ خروجی `.exe` و `.AppImage` |

نتیجه: از لپ‌تاپ یا گوشی، در شبکهٔ خانگی، به سرور وصل می‌شوید و اطلاعات از همان سرور خوانده می‌شود — همان چیزی که خواسته شده بود.

## معماری

```
                 شبکهٔ خانگی (LAN / localhost)
   ┌───────────────┐        HTTP + توکن نشست        ┌────────────────────────┐
   │  لپ‌تاپ / گوشی │  ───────────────────────────►  │      سرور خانگی        │
   │               │                                │  (فقط Node.js ≥ 22)    │
   │ • مرورگر  یا  │  ◄───────────────────────────  │                        │
   │ • اپ Electron │        JSON (بدون رمز خام)      │  ┌──────────────────┐  │
   └───────────────┘                                │  │ Express-less HTTP │  │
                                                    │  │  + Router سبک     │  │
                                                    │  ├──────────────────┤  │
                                                    │  │ Vault (Master PW) │  │
                                                    │  │ AES-256-GCM       │  │
                                                    │  ├──────────────────┤  │
                                                    │  │ SQLite (vault.db) │  │
                                                    │  │ فایل‌ها روی دیسک   │  │
                                                    │  └──────────────────┘  │
                                                    └────────────────────────┘
```

**مدل امنیتی:** یک «کلید صندوق» تصادفی ۲۵۶ بیتی ساخته می‌شود که همهٔ مقادیر حساس با آن رمز می‌شوند. خودِ این کلید با کلیدی که از رمز اصلی مشتق می‌شود (scrypt) رمز و ذخیره می‌شود. **رمز اصلی هرگز ذخیره نمی‌شود.** کلید صندوق فقط در حافظهٔ سرور و فقط تا وقتی نشست باز است نگه داشته می‌شود (**قفل خودکار**). با تغییر رمز اصلی، داده‌ها دوباره رمز نمی‌شوند — فقط کلید صندوق با کلید جدید بسته‌بندی می‌شود (سریع و امن).

## نصب و اجرا

### گام ۱ — اجرای سرور روی سرور خانگی

فقط **Node.js نسخهٔ ۲۲ یا بالاتر** لازم است ([دانلود](https://nodejs.org)). هیچ `npm install` لازم نیست.

**لینوکس:**
```bash
cd webvault-manager/server
./start-linux.sh
```

**ویندوز:** روی `start-windows.bat` دوبار کلیک کنید.

**یا مستقیم:**
```bash
node src/server.js
```

سرور آدرس‌ها را چاپ می‌کند، مثلاً:
```
روی این کامپیوتر:              http://localhost:4600
از شبکهٔ خانگی (لپ‌تاپ/گوشی):  http://192.168.1.20:4600
```

### گام ۲ — باز کردن برنامه

- **از هر دستگاهی در شبکه:** مرورگر را باز کنید و همان آدرس شبکه را وارد کنید.
- بار اول یک **رمز اصلی** تعیین می‌کنید؛ دفعات بعد با همان رمز قفل باز می‌شود.

### نسخهٔ تک‌فایل (`webvault.html`)

اگر یک فایل HTML واحد می‌خواهید (CSS و JS داخلش)، فایل آمادهٔ **`webvault.html`** در ریشهٔ همین پوشه است. آن را مستقیم در مرورگر باز کنید؛ بار اول **آدرس سرور خانگی** را می‌پرسد (مثل `http://192.168.1.20:4600`)، ذخیره می‌کند و وصل می‌شود.

این فایل از ترکیب `public/index.html` + `styles.css` + `app.js` ساخته می‌شود. بعد از هر تغییر در رابط کاربری، دوباره بسازید:
```bash
cd webvault-manager/server && npm run build
```

### گام ۳ (اختیاری) — اپ دسکتاپ واقعی (`.exe` / `.AppImage`)

اگر «آیکون روی دسکتاپ» می‌خواهید، پوستهٔ Electron را بسازید (این گام روی کامپیوتر توسعه، نه لزوماً سرور، انجام می‌شود و به `npm install` نیاز دارد):

```bash
cd webvault-manager/desktop
npm install
npm start                 # اجرای فوری برای تست
npm run dist:win          # خروجی نصب‌کنندهٔ ویندوز (.exe)
npm run dist:linux        # خروجی AppImage لینوکس
```

اپ بار اول آدرس سرور خانگی را می‌پرسد و ذخیره می‌کند.

## تنظیمات سرور (`.env`)

فایل `.env.example` را به `.env` کپی کنید:

| متغیر | پیش‌فرض | توضیح |
|-------|---------|-------|
| `WV_PORT` | `4600` | پورت سرور |
| `WV_HOST` | `0.0.0.0` | `0.0.0.0` یعنی از کل شبکه در دسترس |
| `WV_DATA_DIR` | `./data` | محل ذخیرهٔ دیتابیس و فایل‌ها |
| `WV_CORS` | `*` | محدودسازی به یک آدرس خاص |

> ⚠️ برای امنیت بیشتر، سرور را فقط در شبکهٔ داخلی نگه دارید و آن را مستقیم روی اینترنت باز نکنید.

## طرح دیتابیس (Schema)

| جدول | توضیح |
|------|-------|
| `meta` | نمک KDF، کلید صندوق رمزشده، تنظیمات |
| `websites` | سایت‌ها (اطلاعات عمومی + فنی) |
| `domains` | دامنه‌ها + تاریخ انقضا + nameservers + Cloudflare |
| `servers` | سرورها + SSH (کلید **رمزنگاری‌شده**) + مشخصات سخت‌افزار |
| `credentials` | صندوق رمزها — مقدار محرمانه با **AES-256-GCM** |
| `backups` | تاریخچهٔ بکاپ (نوع، محل، وضعیت) |
| `clients` | مشتری‌ها + وضعیت پرداخت |
| `contracts` | فایل‌ها و قراردادها (روی دیسک، دسته‌بندی‌شده) |
| `tags` + `entity_tags` | سیستم تگ چند‌موجودیتی |
| `activity_log` | ثبت همهٔ تغییرات (چه کسی چه کرد) |

فیلدهای محرمانه (`credentials.secret_enc`, `servers.ssh_key_enc`) **هرگز** به‌صورت Plain Text ذخیره نمی‌شوند و به‌صورت پیش‌فرض در پاسخ API هم برنمی‌گردند — فقط با درخواست صریح `reveal` رمزگشایی می‌شوند و این کار در Audit Log ثبت می‌شود.

## مسیرهای API (خلاصه)

| متد | مسیر | کار |
|-----|------|-----|
| `GET` | `/api/status` | راه‌اندازی شده؟ قفل است؟ |
| `POST` | `/api/setup` / `/api/unlock` / `/api/lock` | رمز اصلی و نشست |
| `POST` | `/api/change-master` | تغییر رمز اصلی |
| `GET` | `/api/dashboard` | آمار، هشدارها، دامنه‌های نزدیک انقضا |
| `GET/POST/PUT/DELETE` | `/api/websites` | مدیریت سایت‌ها |
| `GET/POST/PUT/DELETE` | `/api/domains` | مدیریت دامنه‌ها |
| `GET/POST/PUT/DELETE` | `/api/servers` | مدیریت سرورها |
| `POST` | `/api/servers/:id/test` | تست اتصال (TCP به پورت SSH) |
| `GET/POST/PUT/DELETE` | `/api/credentials` | صندوق رمزها |
| `GET` | `/api/credentials/:id/reveal` | فقط مقدار محرمانه (برای کپی) |
| `POST` | `/api/generate-password` | تولید رمز قوی |
| `GET/POST/PUT/DELETE` | `/api/backups` | بکاپ‌ها |
| `GET/POST/PUT/DELETE` | `/api/clients` | مشتری‌ها |
| `GET/POST/DELETE` | `/api/files` | آپلود/دانلود فایل و قرارداد |
| `GET` | `/api/tags` | همهٔ تگ‌ها + تعداد |
| `GET` | `/api/search?q=` | جستجوی سراسری |
| `GET` | `/api/activity` | فهرست فعالیت‌ها |
| `GET` | `/api/export/json` / `/api/export/csv/:table` | خروجی‌گیری |

همهٔ مسیرها (به‌جز `status`, `setup`, `unlock`) به توکن نشست معتبر در هدر `Authorization: Bearer <token>` نیاز دارند.

## امکانات پیاده‌سازی‌شده

- ✅ **داشبورد:** کل/فعال/فروخته‌شده/در حال توسعه، دامنه‌های نزدیک انقضا، آخرین بکاپ‌ها، هشدارهای امنیتی، درآمد فروش
- ✅ **سایت‌ها:** اطلاعات عمومی + فنی کامل
- ✅ **دامنه‌ها:** با شمارش خودکار روزهای مانده تا انقضا و هشدار
- ✅ **سرورها:** مشخصات کامل + SSH Key رمزشده + **تست اتصال زنده**
- ✅ **صندوق رمزها:** AES-256، انواع (رمز/SSH/API/FTP/DB/ادمین)، **کپی بدون نمایش**، **تولید رمز**، قفل خودکار
- ✅ **بکاپ‌ها:** ثبت محل/نوع/وضعیت/تاریخچه
- ✅ **مشتری‌ها:** با وضعیت پرداخت
- ✅ **فایل منیجر:** آپلود/دانلود، دسته‌بندی (قرارداد/مدارک/تصویر/دسترسی/بکاپ)
- ✅ **جستجوی سراسری:** سایت، دامنه، مشتری، IP، نام کاربری
- ✅ **سیستم تگ:** `#wordpress`, `#sold`, `#urgent`, …
- ✅ **امنیت:** رمزنگاری، Master Password، قفل خودکار، **Audit Log**، بدون ذخیرهٔ Plain Text
- ✅ **خروجی:** JSON و CSV (+ بکاپ کامل برنامه)
- ✅ **رابط کاربری:** Dark Mode، حرفه‌ای، ریسپانسیو، RTL
- ✅ **اپ دسکتاپ:** پوستهٔ Electron با خروجی `.exe` و `.AppImage`

### قابل توسعه (نقطهٔ شروع آماده است)

- زمان‌بندی خودکار بکاپ و اتصال مستقیم به NAS (رکورد و ساختار آماده؛ اجرای زمان‌بند را می‌توان با `cron`/سرویس افزود).
- اجرای واقعی دستور SSH و نمایش زندهٔ مصرف منابع (اکنون **تست اتصال TCP** انجام می‌شود که بدون وابستگی و امن است).
- خروجی PDF (اکنون از «چاپ → ذخیره به‌صورت PDF» مرورگر برای همان نتیجه استفاده کنید).

## پشتیبان‌گیری از خود برنامه

کل داده‌ها در `server/data/` است:
- `vault.db` — دیتابیس (فیلدهای محرمانه رمزشده)
- `files/` — فایل‌ها و قراردادهای آپلودشده

برای بکاپ کامل کافی است از پوشهٔ `data/` کپی بگیرید. برای بکاپ قابل‌حمل، از **تنظیمات → خروجی JSON** استفاده کنید.
